NIO.2 Path Basics

  • `Path` is immutable and platform-independent—build with `Paths.get` or `Path.of`
  • Use `resolve`, `resolveSibling`, and `relativize` to combine or compare paths
  • `normalize` removes redundant `.` and `..` segments without touching the filesystem
  • `toAbsolutePath` converts relative paths relative to the current working directory

Path operations in FileSystemExamples.java

Working with Files

  • `Files.exists`, `isDirectory`, `isSymbolicLink` check metadata without throwing
  • Create resources with `createFile`, `createDirectory`, or `createDirectories` for nested paths
  • `Files.copy`, `move`, and `delete` manage file transfers; specify `StandardCopyOption.REPLACE_EXISTING` to overwrite
  • Use `Files.list` for shallow directory listings; close the resulting stream or use try-with-resources

Filesystem helpers in FileSystemExamples.java

Walking Directory Trees

  • `Files.walk` provides a lazy stream of paths; limit depth or filter as needed
  • `Files.walkFileTree` with `SimpleFileVisitor` allows fine-grained callbacks for visit events
  • Use visitors for recursive copy/delete operations; honour `FileVisitResult` to continue, skip, or terminate
  • Handle symbolic links carefully to avoid cycles—configure `FileVisitOption.FOLLOW_LINKS` explicitly

Stream & Reader/Writer I/O

Byte streams
  • `InputStream`/`OutputStream` transfer raw bytes; wrap with `BufferedInputStream` for performance
  • `Files.newInputStream`/`newOutputStream` bridge NIO.2 paths with classic streams
Character streams
  • `Reader`/`Writer` handle character encodings; prefer `Files.newBufferedReader` for text
  • `PrintWriter` offers convenient formatted output; remember to flush or close

Stream usage in StreamIOExamples.java

Buffered I/O & Copying

  • Buffering reduces disk access frequency, improving throughput
  • `Files.copy` supports stream-to-stream copying; use try-with-resources to close streams automatically
  • `Files.readAllLines` and `readString` load entire files—prefer buffered streaming for large data
  • Combine buffered readers/writers with `lines()` or `write` for concise operations

Buffered operations in StreamIOExamples.java

Attributes & Metadata

  • `Files.getLastModifiedTime`, `size`, `getOwner` expose basic metadata
  • `Files.readAttributes(path, BasicFileAttributes.class)` retrieves a snapshot of attributes
  • POSIX environments offer additional attributes (`PosixFileAttributes`, permissions)
  • Update attributes with `setLastModifiedTime`, `setAttribute`, or `setPosixFilePermissions`

Metadata helpers in Nio2AttributesExamples.java

Serialization

  • Use `ObjectOutputStream`/`ObjectInputStream` to serialize Java objects that implement `Serializable`
  • Provide `serialVersionUID` to maintain compatibility across versions
  • Mark sensitive or derived fields `transient` to exclude them
  • Handle `ClassNotFoundException` and `InvalidClassException` when reading back data

Object stream usage in SerializationExamples.java

Key Takeaways

  • NIO.2 `Path` and `Files` simplify cross-platform filesystem access
  • Buffered streams and readers/writers boost I/O efficiency and reduce boilerplate
  • Attribute APIs expose rich metadata—use them for audits, syncing, or permission handling
  • Serialize cautiously, controlling version compatibility and excluding sensitive fields
