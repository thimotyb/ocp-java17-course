String Fundamentals

  • Strings are immutable objects; any modifier returns a new instance
  • Three concatenation rules drive exam puzzles: numeric + numeric adds, any `String` operand concatenates, evaluation is strictly left-to-right
  • Text blocks (`"""`) produce `String` values with preserved newlines and incidental indentation
  • `+=` performs concatenation by reassigning the variable with the new `String`

String API Toolkit

  • Indexing starts at 0: `length()`, `charAt()`, and `substring()` respect bounds and throw exceptions when out of range
  • Search helpers (`indexOf`, `contains`, `startsWith`, `endsWith`) return positional data or booleans without mutating the source
  • Case conversion (`toUpperCase`, `toLowerCase`) and replacement methods return new values; capture results to avoid data loss
  • `equals` vs `equalsIgnoreCase` differentiate exact matches from case-insensitive comparisons

Method catalogued in StringExamples.java

Whitespace, Formatting, & Text Blocks

  • `strip`, `stripLeading`, `stripTrailing`, and `trim` each handle whitespace; `strip` understands Unicode
  • `isEmpty` checks length; `isBlank` tolerates whitespace-only strings
  • `indent` adjusts indentation and normalises line endings, while `stripIndent` removes incidental leading spaces
  • `translateEscapes` interprets escape sequences; `String.format`/`formatted` inject values with placeholders (e.g., `%s`, `%d`, `%n`)

StringBuilder in Practice

  • `StringBuilder` is mutable; methods like `append`, `insert`, `delete`, `replace`, and `reverse` mutate in place and return `this` for chaining
  • Capacity grows automatically, but you can pre-size builders to reduce reallocations
  • `delete` and `deleteCharAt` enforce bounds; attempts beyond length raise `StringIndexOutOfBoundsException`
  • `StringBuilder` does not override `equals`; compare via `toString()` when logical equality matters

Array Fundamentals

  • Arrays have fixed length and homogeneous element types; declaration syntax allows brackets after the type or variable
  • Array variables hold references; assigning one array to another alias the same data
  • `length` is a field (no parentheses); index access is zero-based and checked at runtime
  • Array store checks occur at runtime—assigning an incompatible type through an `Object[]` reference throws `ArrayStoreException`

Sorting, Searching, Comparing

  • `Arrays.sort` orders primitives or objects in natural ascending order; strings sort lexicographically
  • `Arrays.binarySearch` requires a sorted array; unsuccessful searches return `-(insertionPoint) - 1`
  • `Arrays.compare` performs lexicographical comparison and returns a negative, zero, or positive result
  • `Arrays.mismatch` finds the first differing index or `-1` when arrays are equal

Iterating & Multidimensional Arrays

  • Nested arrays are arrays of arrays; lengths may vary per dimension
  • Initialise with literals (`int[][] grid = {{1,2},{3,4}};`) or create outer arrays before inner ones
  • Enhanced for-loops simplify traversal but expose elements as copies for primitives and references for objects
  • Populate arrays in loops using indices to avoid `NullPointerException` on uninitialised inner arrays

Creating Date & Time Objects

  • `LocalDate`, `LocalTime`, and `LocalDateTime` use factory methods (`of`) with 1-based months
  • `ZonedDateTime.of` combines date, time, and a `ZoneId`; conversions respect daylight saving transitions
  • Most factory methods throw exceptions when supplied invalid values (e.g., day 32 or hour 25)
  • Date/time objects are immutable; assign results from `plus`/`minus` operations

Creations demonstrated in DateTimeCreationExamples.java

Working with Periods & Durations

  • `Period` models date-based amounts; `Duration` models time-based amounts down to nanoseconds
  • Only use `Period` with date-based types (`LocalDate`, `LocalDateTime`); applying to `LocalTime` throws `UnsupportedTemporalTypeException`
  • `Duration` pairs with time-based types (`LocalTime`, `LocalDateTime`); adding to `LocalDate` triggers an exception
  • Builder methods (`ofYears`, `ofMonths`, `ofDays`, `ofHours`, etc.) overwrite previous values when chained—store each result separately

Constraints covered in DateTimePeriodExamples.java and DateTimeDurationExamples.java

Measuring & Adjusting Time

  • `ChronoUnit.between` calculates differences and returns a long in the requested unit (hours, minutes, etc.)
  • `plus`/`minus` operate on immutable date/time objects and must be reassigned to keep the result
  • Combining periods and durations offers fine-grained schedules (e.g., monthly cadence with daily offsets)
  • Always confirm timezone context when mixing `LocalDateTime` with `ZoneId` to avoid unexpected offsets

Difference calculations in DateTimeDurationExamples.java

Key Takeaways

  • Strings are immutable; master concatenation rules and store method results to keep changes
  • `StringBuilder` enables in-place edits but lacks value equality; convert to `String` when comparing
  • Arrays demand sorted input for `binarySearch` and expose lexicographic helpers through `Arrays`
  • `java.time` separates date-based periods from time-based durations; mixing them incorrectly results in runtime errors
